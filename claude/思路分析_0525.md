## 1. 项目概述

### 1.1 设计→仿真架构

```
┌─────────────────────────────────────────────────────────┐
│                  离线设计阶段 (MATLAB)                    │
│                                                         │
│  非线性模型 f(x,u)                                       │
│       │                                                 │
│       ▼ 配平 (fminsearch)                                │
│  平衡点 (x₀,u₀)                                         │
│       │                                                 │
│       ▼ 线性化 (linmod, 数值扰动)                         │
│  线性模型 A₁₃ₓ₁₃, B₁₃ₓ₄                                  │
│       │                                                 │
│       ▼ 提取纵向子空间                                    │
│  A_long(5×5), B_long(5×2)  ← 只用这部分设计控制器         │
│       │                                                 │
│       ▼ 极点配置 / LQR                                   │
│  K_long(2×5)                                            │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                  在线仿真阶段 (Simulink)                   │
│                                                         │
│  K_long ──→ 控制律 u = u_trim + K·(x_ref - x)           │
│                  │                                      │
│                  ▼                                      │
│  ┌──────────────────────────────┐                       │
│  │  F16_dyn MEX (非线性, 13阶)  │  ← 控制的是这个        │
│  │  ẋ = f(x,u)                  │                       │
│  │  不是 A·x + B·u              │                       │
│  └──────────────────────────────┘                       │
└─────────────────────────────────────────────────────────┘
```

**关键事实**：$K$ 是用线性模型（5 阶纵向子空间）设计的，但实际施加在非线性模型（13 阶全状态）上。线性模型只是非线性模型在配平点的局部一阶近似：

$$f(x_0+\Delta x, u_0+\Delta u) \approx A\Delta x + B\Delta u$$

当 $(\Delta x, \Delta u)$ 很小时（如 $\Delta\theta=1°$），这个近似通常足够好。但当飞机做大机动时（$\Delta\theta$ 几十度），线性近似失效——这就是为什么 `place()` 算出的 $K$ 在实际非线性仿真中可能表现不佳。

### 1.2 当前已实现：纵向 + 横侧向分通道

### 1.3 理论最优 vs 工程可行：为什么可以手动调整 K

`place()` 和 `lqr()` 给出的 $K$ 是**针对线性模型的最优解**。但实际控制器跑在非线性模型上，两者之间存在"模型失配"（Model Mismatch）。

**place/lqr 的最优性前提**：

- `place(A, B, p)` 保证闭环极点精确落在指定位置 $p$——但前提是 $A, B$ 完全准确
- `lqr(A, B, Q, R)` 在二次代价 $J=\int(x^TQx+u^TRu)dt$ 下最优——但代价函数未必反映真实物理需求
- 两者的最优性都只对**线性模型**成立，对非线性模型只是"在配平点附近的良好近似"

**HIFI 模型暴露的问题**：

lab3_0525_2 的经验表明，LOFI 下 `place` 算出的 $K_{2,3}=-4.6$（升降舵对俯仰角的增益），切换到 HIFI 后仿真在 ~0.18s 崩溃。手动将 $K_{2,3}$ 降到 -0.8 后 HIFI 才稳定。

**原因**：HIFI 含前缘襟翼（dLEF）自动偏转，气动表的非线性更强。大增益在 LOFI 的"平滑"气动曲线上没问题，但在 HIFI 的"粗糙"曲线上引起舵面振荡→发散。

**手动调 K 的本质**：

手动降增益不是"放弃最优"，而是**鲁棒性降级**（Robustness Degradation）：

| 维度 | place/lqr | 手动降增益 |
|------|-----------|-----------|
| 线性模型精度 | 最优 | 次优 |
| 非线性鲁棒性 | 可能崩溃 | 通常稳定 |
| 适用范围 | 小扰动 | 更大机动范围 |
| 工程判断 | 无需 | 需要物理直觉 |

**标准做法**：先用 place/lqr 设计 $K$（起点），再根据非线性仿真反馈手动微调（迭代）。这是航空航天工程的常规流程，不是"凑数"。

## 2. 状态空间与控制架构

### 2.1 13 维全状态 vs 10 维动力学状态

13 维全状态 = $[V, \beta, \alpha, \phi, \theta, \psi, p, q, r, x_E, y_E, h, P_{ow}]$

差的 3 个是**位置变量**：$x_E$（北向）、$y_E$（东向）、$h$（高度）。它们是运动学积分：

$$\dot{x}_E = V\cos\theta\cos\psi, \quad \dot{y}_E = V\cos\theta\sin\psi, \quad \dot{h} = V\sin\theta$$

位置**不产生气动力/力矩**，不影响飞机动力学。控制器设计只需要 10 个动力学状态，3 个位置变量留给制导层（Phase 4）。

### 2.2 纵横向分块 vs 联合矩阵

**问题**：Phase 3 联合闭环时，是否需要分别用 $(A_{lon}, B_{lon})$ 和 $(A_{lat}, B_{lat})$ 两套矩阵设计两个 $K$？

**答案**：可以用一个合并的 $A$ (10×10) 和 $B$ (10×4) 一次性设计。

**合并方法**：

从 13×13 全状态 A 矩阵中，按 $[V, \alpha, \theta, q, P_{ow}, \beta, \phi, \psi, p, r]$ 顺序重新排列，提取 10×10 子矩阵：

$$A_{combined} = \begin{bmatrix} A_{lon}(5\times5) & A_{cross1}(5\times5) \\ A_{cross2}(5\times5) & A_{lat}(5\times5) \end{bmatrix}$$

其中 $A_{cross1}$ 和 $A_{cross2}$ 是纵横向交叉耦合项（如滚转引起的 $\alpha$ 变化、俯仰引起的 $\beta$ 变化）。

**为什么可以先分后合**：

在配平点附近，交叉耦合项 $A_{cross}$ 通常比对角块 $A_{lon}$、$A_{lat}$ 小 1-2 个数量级。因此先分别设计 $K_{lon}$ 和 $K_{lat}$，再组合，效果通常接近联合设计。但如果需要精确解耦或做大机动，联合设计更可靠。

### 2.3 级联控制架构

飞机控制采用**级联（Cascade）控制**：外环慢，内环快。

```
外环（制导） — 慢，响应 ~10-30s
  "飞到 H=6000m"    → θ_cmd = K_h·(h_ref - h)
  "右转航向 90°"    → φ_cmd = K_ψ·(ψ_ref - ψ)
  "保持 V=200 m/s"  → δth_cmd = K_V·(V_ref - V)
        │
        ▼  θ_cmd, φ_cmd, δth_cmd 作为内环的参考输入
内环（姿态控制） — 快，响应 ~0.5-1s
  θ_cmd → δe    （K_lon，我们已实现）
  φ_cmd → δa,δr （K_lat，我们已实现）
        │
        ▼  舵面指令
   飞机动力学（F16_dyn）
```

**为什么必须先内环后外环**：

内环不稳的话，外环下达的 $\theta_{cmd}$ 根本执行不了——飞机还没来得及跟踪 $\theta_{cmd}$，外环又发了新指令，系统发散。内环必须先能快速、准确地跟踪姿态指令，外环才有意义。

**当前状态**：
- 内环（Phase 1-3）：纵向 θ 跟踪 ✅，横向 φ 跟踪 ✅，联合闭环待做
- 外环（Phase 4）：高度/航向/速度制导，待做

## 4. 模态分析与控制策略

### 4.1 纵向模态

纵向线性模型（5 阶）包含两个主要模态：

**短周期 (Short Period)** — 快速模态：
- 特征值为一对共轭复数，$\omega_n$ 较大，响应时间 ~0.5-1s
- 主要涉及 $\alpha$（迎角）和 $q$（俯仰角速率）的耦合振荡
- 飞行员感受到的"抬头/低头"就是短周期模态
- 控制目标：通过升降舵 $\delta_e$ 调节阻尼和频率

**长周期 / Phugoid** — 慢速模态：
- 特征值为一对共轭复数，阻尼很小，周期 ~30-60s
- 主要涉及 $V$（速度）和 $\theta$（俯仰角）的交换：飞机抬头→减速→下沉→俯冲→加速→再抬头
- 能量守恒过程，动能和势能交替转换
- 控制目标：通过油门 $\delta_{th}$ 和升降舵联合抑制

**为什么纵向控制需要两个输入**：
- $\delta_e$（升降舵）：主要控制短周期——快速改变俯仰姿态
- $\delta_{th}$（油门）：主要控制长周期——通过推力改变能量状态
- 两者协同才能同时控制姿态和速度

### 4.2 横侧向模态

**LOFI 横侧向**特征值：$\lambda = \{0,\; -0.29\pm3.01i\;(\text{荷兰滚}),\; -2.43\;(\text{滚转收敛}),\; -0.010\;(\text{螺旋})\}$

荷兰滚 (Dutch Roll) — $\lambda = -0.29\pm3.01i$：
- $\omega_n=3.02$ rad/s，$\zeta=0.095$（阻尼很低！）
- 侧滑 $\beta$ 和偏航 $r$ 之间的耦合振荡
- 类似纵向的短周期，但涉及横侧向自由度

滚转收敛 (Roll Subsidence) — $\lambda = -2.43$：
- $\tau=0.41$ s，快速收敛
- 副翼偏转后滚转速率 $p$ 的指数衰减

螺旋模态 (Spiral) — $\lambda = -0.010$：
- 接近临界稳定，$\tau=101$ s
- 微小扰动引起的缓慢滚转-偏航发散
- 几乎所有飞机都需要反馈来稳定螺旋模态

航向模态 (Heading) — $\lambda = 0$：
- 一阶积分器，无自然恢复
- $\dot{\psi} = r$，需要外部反馈维持航向

### 4.3 横侧向控制策略

横侧向需要**两个控制回路**：

**滚转回路**（通过副翼 $\delta_a$）：
$$\phi_{cmd} \to \text{滚转控制器} \to \delta_a \to p \to \phi$$

**偏航回路**（通过方向舵 $\delta_r$）：
$$\beta \to 0 \text{ (协调转弯)} \to \text{偏航控制器} \to \delta_r$$

同时还需要一个**ARI (Aileron-Rudder Interconnect)** 来消除副翼偏转产生的 adverse yaw。

## 5. 验证维度与质量评估

### 5.1 纵向验证维度

**仅跟踪俯仰角 $\theta$ 是不够的。** 纵向控制的完整验证需要覆盖以下维度：

| 验证维度 | 物理意义 | 判据 | 当前状态 |
|:--------:|---------|------|:--------:|
| **$\theta$ 跟踪** | 俯仰角能否跟踪指令 | $\|\theta_{ss} - \theta_{ref}\| < 0.5°$ | ✅ lab3_0525_2 已验证 |
| **$V$ 保持** | 扰动后速度能否恢复 | $\|V - V_{trim}\| < 5$ m/s/10s | ⚠️ verify_trim 已测（3 m/s），未单独优化 |
| **$q$ 阻尼** | 俯仰速率不应振荡 | $q$ 无持续振荡 | ❌ 未单独验证 |
| **Phugoid 抑制** | 长周期 $V$-$\theta$ 振荡 | 振幅衰减 > 50%/周期 | ❌ 未测试 |
| **$\alpha$ 保护** | 迎角不接近失速 | $\alpha < \alpha_{max}$ | ❌ 属于 Phase 3 大机动场景 |
| **$h$ 跟踪** | 高度能否稳定 | 需制导外环 | ❌ 属于 Phase 4 |
| **$\delta_e$ 合理性** | 升降舵应在物理限幅内 | $\|\delta_e\| < 25°$ | ❌ 未检查 |

**$V$ 保持的物理意义**：当飞机受到扰动（如阵风）后速度偏离配平值，控制器应能将速度拉回。当前纵向控制器的 $K_{1,:} \approx 0$（油门增益极小），意味着油门几乎不参与反馈，速度恢复主要靠 phugoid 模态的自然阻尼和升降舵的间接调节。

**$h$ 跟踪和 $\alpha$ 保护的时机**：
- **$h$ 跟踪**需要制导外环（$\theta_{cmd} = K_h(h_{ref} - h)$），是 Phase 4 的内容。前提是内环（姿态控制）已稳定
- **$\alpha$ 保护**在小角度机动（$\theta$ 跟踪 3°、$\phi$ 跟踪 5°）时不需要。只有 Phase 3 联合闭环做大机动时才可能接近失速

### 5.2 横侧向验证维度

**仅跟踪滚转角 $\phi$ 是不够的。** 横侧向控制的完整验证需要覆盖以下维度：

| 验证维度 | 物理意义 | 判据 | 当前状态 |
|:--------:|---------|------|:--------:|
| **$\phi$ 跟踪** | 滚转角能否跟踪指令 | $\|\phi_{ss} - \phi_{ref}\| < 0.5°$ | ✅ lab4_0527 已验证 |
| **$\beta$ 抑制** | 侧滑角应接近 0（协调转弯） | $\|\beta\| < 1°$ | ❌ 未单独验证 |
| **$p$ 阻尼** | 滚转速率不应振荡 | $p$ 无持续振荡 | ❌ 未单独验证 |
| **$\psi$ 跟踪** | 偏航角能否稳定 | $\psi$ 漂移 < 5°/30s | ❌ 未单独验证 |
| **$\delta_a, \delta_r$ 合理性** | 舵面偏转应在物理限幅内 | $\|\delta_a\|<21.5°, \|\delta_r\|<30°$ | ❌ 未检查 |

**为什么 $\beta$ 抑制特别重要**：

$\beta$（侧滑角）不为零意味着飞机在"侧着飞"——机身侧面承受气流，阻力剧增，乘客不适，结构载荷不对称。协调转弯（Coordinated Turn）的核心就是 $\beta \approx 0$。当前 lab4_0527 的横侧向控制器用 $K_{lat}$ 同时处理 $\beta, \phi, \psi, p, r$ 五个状态，理论上 $\beta$ 会被驱动到接近零，但**没有单独验证**。

**为什么 $\psi$ 跟踪需要关注**：

横侧向 A 矩阵有一个**零特征值**（航向模态）——$\psi$ 没有自然恢复力。如果控制器不能有效稳定航向，飞机会在扰动后指向任意方向。这在编队飞行、对地攻击等任务中是不可接受的。

### 5.3 LOFI vs HIFI 对比

| 维度 | LOFI | HIFI | 差异来源 |
|------|------|------|---------|
| **气动模型** | 不计前缘襟翼 | 含 dLEF 自动偏转 | HIFI 更接近真实飞机 |
| **配平舵面** | $\delta_e=-1.87°$ | 待测 | HIFI 含 dLEF，改变升力/力矩平衡 |
| **纵向 θ 跟踪** | place: 0.23°, lqr: 0.12° | 待测 | HIFI 非线性更强 |
| **横向 φ 跟踪** | place: 0.38°, lqr: 0.04° | 待测 | HIFI 气动表更精细 |
| **荷兰滚阻尼** | $\zeta=0.095$（很低） | 待测 | HIFI 的 $\beta$ 对力矩的影响可能不同 |
| **表越界风险** | 低 | **高** | 大 $\alpha/\beta$ 时 HIFI 表可能溢出 |
| **仿真稳定性** | 稳定 | 大机动时可能崩溃 | HIFI 需要更保守的增益 |
| **增益敏感度** | 低 | **高** | K(2,3)=-4.6 在 LOFI 正常，HIFI 崩溃 |

**LOFI 适合**：控制器初步设计、快速迭代、模态分析、增益调试
**HIFI 适合**：最终验证、性能评估、接近真实飞机的行为预测

**建议**：先用 LOFI 完成全部验证维度，再切换 HIFI 做最终验证。如果 HIFI 崩溃，降低增益重试。

### 5.4 残差分析：为什么纵向误差远大于横侧向

lab4_0527 的残差分析给出了定量对比：

| 指标 | 纵向 | 横侧向 | 比值 |
|------|------|--------|:----:|
| $\|r\|$ | **2.01** | **0.012** | **165:1** |

纵向残差的主因是 $\dot{V} = -1.99$ m/s²（来自 $X_V V + X_\theta \theta + X_{Pow} P_{ow}$ 的大数互抵不平衡）。横侧向残差极小是因为 $\beta, \phi, \psi$ 在配平点附近没有显著的"大数互抵"项。

**这解释了为什么**：
- 纵向跟踪 $\theta=3°$ 时有明显稳态误差（place: 0.23°, lqr: 0.12°）
- 横侧向跟踪 $\phi=5°$ 时误差极小（place: 0.38°, lqr: 0.04°）
- 纵向需要积分控制（PI）来消除残差，横侧向通常不需要

## 7. 爬升→盘旋→下降 完整机动的实现路线

### 7.1 机动分解

```
爬升 (Climb):
  目标: 从 H=5000m 爬升到 H=6000m
  控制: 增大油门 δth, 调整升降舵 δe 维持速度 V=200m/s
  时间: ~30-60s

盘旋 (Turn):
  目标: 水平盘旋 360° (或 180°)
  控制: 副翼 δa 产生滚转角 φ≈30°, 配合方向舵 δr 协调转弯
        升降舵 δe 补偿升力损失 (维持高度)
        油门 δth 补偿转弯阻力 (维持速度)
  时间: ~30-60s (取决于转弯速率)

下降 (Descend):
  目标: 从 H=6000m 下降到 H=5000m
  控制: 减小油门 δth, 调整升降舵 δe 维持速度
  时间: ~30-60s
```

### 7.2 控制架构

```
                    ┌─────────────────────┐
                    │   制导指令生成器      │
                    │  H_ref(t), ψ_ref(t)  │
                    │  V_ref = 200 m/s     │
                    └──────────┬──────────┘
                               │
            ┌──────────────────┼──────────────────┐
            ▼                  ▼                  ▼
    ┌───────────────┐  ┌───────────────┐  ┌───────────────┐
    │  高度控制器     │  │  航向控制器     │  │  速度控制器     │
    │  θ_cmd =       │  │  φ_cmd =       │  │  δth_cmd =     │
    │  K_h(H_ref-H)  │  │  K_ψ(ψ_ref-ψ)  │  │  K_V(V_ref-V)  │
    └───────┬───────┘  └───────┬───────┘  └───────┬───────┘
            │                  │                  │
            ▼                  ▼                  │
```

### 7.3 技术难点预估

| 难点 | 说明 | 解决方向 |
|------|------|---------|
| **纵横向耦合** | 滚转时升力矢量倾斜 → 需要增大 α 维持高度 → δe 需补偿 | ARI + 协调转弯配平 |
| **HIFI 表越界** | 大机动时 α/β 更易超出表格范围 | 在 C 代码中钳位 α,β |
| **非线性效应** | 大滚转角 (φ>30°) 时线性模型精度急剧下降 | 增益调度 (Gain Scheduling) |
| **速度控制** | 油门-速度通道有发动机延迟 (~1s) | 前馈 + PI 控制 |
| **配平点切换** | 爬升/盘旋/下降各有不同的配平状态 | 需要多配平点 (trim map) |
